What is this Documentation Template? Glad you're reading this documentation! 😄 This is a template for creating Documentations. The main objective of this template is to make life easier for anyone who needs to create documentation, whether for a job, project, developed software, APIs, etc. This template uses pure JavaScript to provide new HTML Tags that make life easier and speed up the creation of documentation for anyone with HTML knowledge to edit this template!

PS: Do not be fooled by the size of this documentation, it is very easy to use this Documentation Template! 😉 Using the Documentation Template To use this Documentation Template is very simple. You can download it through the official GitHub and then, just edit the document called "Documentation.html". To edit it is really very simple, you just need to edit the content that is within the commented area with START OF SUMMARY CONTENT, END OF SUMMARY CONTENT and START OF DOCUMENTATION CONTENT, END OF DOCUMENTATION CONTENT as the example below shows! This template is also responsive and offers support for mobile devices. ...HTML code... EDIT HERE ...HTML code... EDIT HERE TOO ...HTML code... Use HTML tags with doc. prefix (HTML tags introduced by this template) only in the content of your documentation, that is, between the START OF DOCUMENTATION CONTENT and END OF DOCUMENTATION CONTENT comment tags. Prerequisites for using Documentation Template This Documentation Template has some small prerequisites for it to be usable. See the list below for these prerequisites. The browser of the user who opens this documentation must support JavaScript and have JavaScript enabled. This Documentation Template depends heavily on JavaScript to pre-process the page completely, before showing it to the user, processing all tags and items in this documentation. This template was developed to be compatible with any browser that uses Blink/Webkit engines (Edge, Chrome, Safari, Opera, etc) and Mozilla Firefox. (Optional) You must use VS Code to edit this template. VS Code has several tools that can greatly facilitate the editing of this template, such as auto code completion, automatic identation, hiding parts of the code using <!-- region --> tags and much more. Worth a try! (Optional) If you use the VS Code to edit this template, You can download the Path Autocomplete extension to have the auto complete for file paths working in the src attributes of the doc tags of this template. (Optional) If you use the VS Code to edit this template, You can download the Text Marker extension to highlight the => symbols that are used in the tables in this template.
