What is this Documentation Template? Glad you're reading this documentation! üòÑ This is a template for creating Documentations. The main objective of this template is to make life easier for anyone who needs to create documentation, whether for a job, project, developed software, APIs, etc. This template uses pure JavaScript to provide new HTML Tags that make life easier and speed up the creation of documentation for anyone with HTML knowledge to edit this template!

PS: Do not be fooled by the size of this documentation, it is very easy to use this Documentation Template! üòâ Using the Documentation Template To use this Documentation Template is very simple. You can download it through the official GitHub and then, just edit the document called "Documentation.html". To edit it is really very simple, you just need to edit the content that is within the commented area with START OF SUMMARY CONTENT, END OF SUMMARY CONTENT and START OF DOCUMENTATION CONTENT, END OF DOCUMENTATION CONTENT as the example below shows! Use HTML tags with doc. prefix (HTML tags introduced by this template) only in the content of your documentation, that is, between the START OF DOCUMENTATION CONTENT and END OF DOCUMENTATION CONTENT comment tags. Prerequisites for using Documentation Template This Documentation Template has some small prerequisites for it to be usable. See the list below for these prerequisites. The browser of the user who opens this documentation must support JavaScript and have JavaScript enabled. This Documentation Template depends heavily on JavaScript to pre-process the page completely, before showing it to the user, processing all tags and items in this documentation. This template was developed to be compatible with any browser that uses Blink/Webkit engines (Edge, Chrome, Safari, Opera, etc) and Mozilla Firefox. (Optional) You must use VS Code to edit this template. VS Code has several tools that can greatly facilitate the editing of this template, such as auto code completion, automatic identation, hiding parts of the code using <!-- region --> tags and much more. Worth a try! Creating multi-page documentation with this Template Sometimes it may be necessary to create documentation with several HTML pages. In such cases, you can simply duplicate the "Documentation.html" file, generating a "Documentation-PartTwo.html" for example, and then editing its content. See the example below if you still have questions! Just don't forget to keep both HTML pages in the same directory, okay? The <doc.code> tag has some characteristics... The < and > characters are formatted for display automatically. You do not need to use the &lt; and &gt; symbols to display < and > respectively. If your code is not an HTML/PHP/JavaScript code, and you need to use < and >, the tag will prevent a closed tag from being created automatically by the browser. So, you can use the C# var variable = object.GetComponent<Type>();code without problems. The tag will also remove unnecessary line breaks that are before or after your code between <doc.code> tags.
